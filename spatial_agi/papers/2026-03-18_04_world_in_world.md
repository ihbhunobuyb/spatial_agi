## 1. 论文概述

### 1.1 研究背景

生成式世界模型（World Models, WMs）在视觉生成领域取得了显著进展，能够以惊人的视觉真实感模拟世界。这一技术进步自然引发了一个关键问题：**世界模型能否为具身智能体（embodied agents）提供预测感知能力，从而辅助决策？**

然而，当前研究面临一个核心挑战：**评估协议的碎片化**。大多数现有基准测试采用**开环协议（open-loop protocols）**，仅孤立地强调视觉质量，而忽视了具身实用性的核心问题——即世界模型是否真正能够帮助智能体在具身任务中取得成功。

### 1.2 核心贡献

本文提出了**World-in-World**，这是第一个在闭环世界中评估世界模型的开放平台，其核心特点包括：

1. **统一的在线规划策略**：提出-模拟-修订的三阶段闭环规划框架
2. **标准化的动作API**：支持异构世界模型的集成与评估
3. **四个闭环具身环境**：Active Recognition、Image-Goal Navigation、Active Embodied QA、Robotic Manipulation
4. **首个世界模型的数据Scaling Law**：研究后训练数据规模对任务成功的影响

### 1.3 三大惊人发现

通过系统性的实验，本文揭示了三个令人惊讶的发现：

| 发现 | 描述 |
|------|------|
| **发现1** | 视觉质量 alone 不能保证任务成功——**可控性（controllability）更为重要** |
| **发现2** | 使用动作-观察数据进行后训练扩展，比升级预训练视频生成器**更有效** |
| **发现3** | 分配更多**推理时间计算资源**可以显著提升闭环性能 |

## 2. 核心算法原理

### 2.1 闭环在线规划框架

World-in-World的核心是一个统一的闭环在线规划策略，其工作流程包含三个阶段：

```
┌─────────────────────────────────────────────────────────────────┐
│                    闭环在线规划流程                               │
├─────────────────────────────────────────────────────────────────┤
│  时间步 t:                                                      │
│                                                                 │
│  1. Proposal (提议阶段):                                        │
│     ┌─────────────────┐                                        │
│     │  π_proposal     │  → 生成 M 个候选动作计划               │
│     │  (提议策略)      │    {Â^{(1)}_t, Â^{(2)}_t, ..., Â^{(M)}_t} │
│     └─────────────────┘                                        │
│           ↓                                                     │
│  2. Simulation (模拟阶段):                                      │
│     ┌─────────────────┐                                        │
│     │  World Model    │  → 预测每个候选计划的未来观察           │
│     │  g_θ            │    {Ô^{(1)}_t, Ô^{(2)}_t, ..., Ô^{(M)}_t} │
│     └─────────────────┘                                        │
│           ↓                                                     │
│  3. Revision (修订阶段):                                        │
│     ┌─────────────────┐                                        │
│     │  π_revision     │  → 评估所有rollout，选择最优决策        │
│     │  (修订策略)      │    D*_t = argmax S(Â^{(m)}_t, Ô^{(m)}_t) │
│     └─────────────────┘                                        │
│           ↓                                                     │
│  执行 D*_t 于实际环境，获得新观察 o_{t+1}                        │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 数学形式化

#### 2.2.1 提议阶段

给定当前观察 $o_t$ 和任务目标 $g$，提议策略 $\pi_{proposal}$ 采样 $M$ 个候选动作序列：

$$\hat{A}^{(m)}_t \sim \pi_{proposal}(A|o_t, g), \quad m = 1, ..., M$$

#### 2.2.2 模拟阶段

每个候选计划通过统一动作API转换为世界模型的控制输入：

$$I^{(m)}_t = C(\hat{A}^{(m)}_t)$$

然后，世界模型 $g_\theta$ 执行反事实 rollout，预测未来世界状态：

$$\hat{O}^{(m)}_t \sim g_\theta(O|o_t, I^{(m)}_t), \quad \hat{O}^{(m)}_t = \{\hat{o}^{(m)}_{t+1}, \hat{o}^{(m)}_{t+2}, ..., \hat{o}^{(m)}_{t+L}\}$$

其中 $L$ 是预测视界。

#### 2.2.3 修订阶段

修订策略评估所有rollout并选择最大化期望奖励的决策：

$$D^*_t = \pi_{revision}(\{(\hat{A}^{(m)}_t, \hat{O}^{(m)}_t)\}_{m=1}^M, o_t, g)$$

最常见的实例化是将 $\pi_{revision}$ 实现为 score-and-select 算子 $S$：

$$m^* = \arg\max_{m \in \{1,...,M\}} S(\hat{A}^{(m)}_t, \hat{O}^{(m)}_t|o_t, g), \quad D^*_t = \hat{A}^{(m^*)}_t$$

### 2.3 统一动作API

为了使不同类型的世界模型能够无缝集成，论文提出了统一动作API，支持三种主要的控制信息类型：

#### 2.3.1 文本提示（Text Prompt）

对于图像到视频的世界模型，控制器将意图动作序列映射为描述性文本提示。使用预定义模板将每个原始动作转换为短语，然后连接这些短语生成最终提示 $I_{text}$。

#### 2.3.2 相机轨迹/视点（Camera Trajectory / Viewpoint）

对于消耗显式视点的模型，控制器将动作序列转换为相机轨迹。例如，每个平移动作将相机移动0.2m，每个旋转动作改变方位角22.5°。轨迹表示为序列 $\{(x_k, y_k, \phi_k)\}_{k=1}^K$。

#### 2.3.3 低级动作（Low-level Actions）

对于以离散或连续低级动作作为输入的世界模型，控制器将动作序列映射到世界模型的动作词汇表，建立智能体动作与世界模型输入之间的对应关系。

### 2.4 后训练方案

为了评估将预训练视频生成器适配为具身世界模型的可行性，论文引入了一个后训练程序，使预训练模型与目标环境领域分布和动作空间对齐。

#### 2.4.1 后训练目标

给定初始RGB帧 $x_1 \in \mathbb{R}^{3 \times H \times W}$，目标合成 $N$ 帧视频 $X = \{x_1, x_2, ..., x_N\}$，表示执行动作序列 $A = \{a_1, a_2, ..., a_N\}$ 后的未来观察序列。

后训练学习条件分布 $p_\theta(X|x_1, C(A))$，其中 $C(A)$ 是统一动作API发出的控制信号。

#### 2.4.2 数据集构建

- **Habitat-Sim任务**（AR, A-EQA, ImageNav）：在HM3D训练场景上收集的全景动作-观察数据集
- **Robotic Manipulation任务**：使用RLBench生成的任务演示

**关键点**：用于后训练的所有Habitat-Sim数据来自与评估场景不相交的场景，确保评估场景对世界模型来说是** unseen**的。

## 3. 与Spatial AGI的关系

### 3.1 Spatial AGI的定义与内涵

Spatial AGI（空间通用人工智能）是指能够在三维物理空间中感知、理解、推理和行动的人工智能系统。其核心特征包括：

1. **空间感知能力**：理解物体在三维空间中的位置、姿态和关系
2. **动态预测能力**：预测动作对环境状态的影响
3. **闭环决策能力**：基于观察反馈持续调整行为
4. **跨域泛化能力**：在新环境中有效执行任务

### 3.2 World-in-World对Spatial AGI的贡献

#### 3.2.1 闭环评估范式

World-in-World的核心贡献在于将世界模型的评估从**开环**转变为**闭环**。这与Spatial AGI的核心需求高度一致：

| 开环评估 | 闭环评估 |
|---------|---------|
| 孤立的视觉质量评估 | 任务成功率作为主要指标 |
| 忽视动作-观察交互 | 模拟真实智能体-环境交互 |
| 无法衡量实际效用 | 直接衡量对决策的贡献 |

#### 3.2.2 可控性作为关键因素

论文发现**可控性（controllability）比视觉质量更重要**。这与Spatial AGI的关系在于：

- **精确的动作控制**是实现空间智能的基础
- 世界模型需要准确响应动作指令，才能为规划提供可靠预测
- 可控性通过1-LPIPS（预测观察与真实观察之间的距离）量化

#### 3.2.3 数据Scaling Law的启示

论文提出的首个世界模型数据Scaling Law表明：

- **动作条件的后训练**比升级预训练视频生成器更有效
- 更大的后训练数据集带来持续的改进
- 这为Spatial AGI的数据收集和训练策略提供了重要参考

### 3.3 核心算法与Spatial AGI的关联

#### 3.3.1 在线规划框架

论文提出的"提议-模拟-修订"框架直接对应Spatial AGI的决策循环：

1. **提议**：基于当前状态和目标生成候选计划
2. **模拟**：利用世界模型预测动作后果
3. **修订**：评估并选择最优决策
4. **执行**：在真实环境中执行并观察结果
5. **迭代**：重复上述过程

这一框架与人类的空间推理和决策过程高度相似，是实现Spatial AGI的重要里程碑。

#### 3.3.2 多任务评估

World-in-World涵盖的四个任务覆盖了Spatial AGI的关键能力：

| 任务 | Spatial AGI能力 |
|------|---------------|
| Active Recognition | 空间感知、目标识别、主动探索 |
| Image-Goal Navigation | 视觉导航、空间定位、路径规划 |
| Active Embodied QA | 空间推理、问答、主动探索 |
| Robotic Manipulation | 精确操作、物理交互、闭环控制 |

## 4. 主要创新点

### 4.1 创新点一：首个闭环世界模型基准

**传统方法**：
- VBench：评估视频生成质量（美学、图像质量等）
- WorldModelBench：评估视觉 plausibility
- WorldScore：评估接受图像和相机轨迹输入的模型

**World-in-World的创新**：
- 第一个测试生成世界是否真正增强具身推理和任务性能的基准
- 强调**任务成功率**而非孤立视觉质量
- 模拟真实智能体-环境交互的闭环设置

### 4.2 创新点二：统一的动作API和规划框架

**技术创新**：
- 提出标准化的动作API，支持三种控制类型：文本提示、相机轨迹、低级动作
- 设计统一的在线规划策略，使异构世界模型可在同一框架下评估
- 实现"提议-模拟-修订"的三阶段闭环决策流程

**实际价值**：
- 任何世界模型可以通过适配器接入系统
- 公平比较不同架构的世界模型
- 为后续研究提供可扩展的评估平台

### 4.3 创新点三：数据Scaling Law

**发现**：
- 后训练数据规模与任务成功率呈正相关
- 40K后训练实例后，Wan2.1†从60.25%提升到63.34%
- 更大的模型从更多数据中获益更多

**意义**：
- 为世界模型的数据收集提供量化指导
- 揭示了动作条件训练的重要性
- 为Spatial AGI的训练策略提供参考

### 4.4 创新点四：推理时间Scaling

**发现**：
- 增加推理时的模拟次数可显著提升成功率
- 平均推理次数从3增加到11，SR从53.36%提升到60.98%

**启示**：
- 推理计算可以弥补模型能力的不足
- 为计算资源分配提供参考
- 开辟了"推理时间训练"的新研究方向

### 4.5 创新点五：可控性优先于视觉质量

**关键洞察**：
- 视觉质量与任务成功率的相关性较弱
- 可控性（动作响应准确性）是成功的更关键因素
- 后训练显著提升可控性并带来相应的性能提升

**对领域的冲击**：
- 挑战了"视觉质量越高越好"的假设
- 为世界模型的发展指明新方向
- 强调功能性评估的重要性

## 5. 局限性分析

### 5.1 泛化能力有限

#### 5.1.1 问题描述

大多数视频生成器在网络视频上进行预训练。在 unseen 的具身环境中，它们可能回归到训练先验或忽略动作控制，产生物理或语义上不一致的rollout。

#### 5.1.2 影响

- 这些偏差会误导规划并降低成功率
- 在新环境中的泛化能力仍然是一个重大挑战

#### 5.1.3 可能的解决方案

- 更大的模型或更多预训练数据
- 统一动作表示
- 课程学习和领域特定数据收集

### 5.2 长视界规划的困难

#### 5.2.1 问题描述

在实验中，视觉世界模型只能模拟短期变化。由于缺乏累积时空历史的机制，长视界任务表现不佳。

#### 5.2.2 影响

- 难以维持场景级上下文
- 无法支持扩展视界内的连贯规划

#### 5.2.3 尝试的方案及效果

- 尝试使用全景输入提供全局上下文
- 增益在模型和任务间不一致
- 原因：全景到透视的转换引入分辨率损失

### 5.3 精确交互和动力学建模困难

#### 5.3.1 问题描述

对于操作任务，捕捉接触丰富的交互、柔顺性、摩擦和关节/可变形物体的状态变化至关重要。当前视觉世界模型经常忽略这些细节，产生违反物理的rollout。

#### 5.3.2 实验证据

 Robotic Manipulation的结果显示：
 - 最佳后训练模型（SVD†）达到46.5% SR，仅比VLM基线（44.5%）略有提升
 - 增益远不如感知和导航任务显著

#### 5.3.3 未来方向

- 物理引导的运动生成
- 推断或生成物理特性以通知动作条件预测

### 5.4 提议和修订策略的性能天花板

#### 5.4.1 问题描述

智能体的整体性能取决于世界模型保真度和提议/修订策略选择和修订决策的强度。

#### 5.4.2 影响

- 基线策略必须有效才能提供可靠的起点
- 强化它们可以提升天花板

#### 5.4.3 可能的改进

- 探索更强的策略
- 深化世界模型和决策之间的协同

### 5.5 评估任务的局限性

#### 5.5.1 任务多样性

虽然涵盖四个任务，但仍有局限性：
- 仅室内场景
- 模拟环境，未涉及真实机器人
- 任务类型可以更丰富

#### 5.5.2 评估指标

- 依赖模拟器的真实性
- 真实世界迁移效果未知

### 5.6 技术实现的其他局限

#### 5.6.1 计算资源

- 后训练需要大量GPU资源
- 推理时间Scaling增加计算成本

#### 5.6.2 模型依赖

- 评估的模型数量有限
- 未评估所有最新的世界模型

### 6.5 视觉质量 vs 任务成功

论文明确指出：
- **视觉质量与任务成功率的相关性很弱**
- 一些视觉质量最高的模型在任务成功率上表现不佳
- 可控性（1-LPIPS）是更重要的指标

这挑战了"视觉质量越高越好"的传统假设。

## 7. 数据 Scaling Law

### 7.1 实验设置

研究后训练数据规模如何影响WM性能：
- 数据规模：400到80K实例
- 评估任务：Active Recognition
- 模型：Wan2.2†, Wan2.1†, SVD†

### 7.2 结果

```
Seen Examples During Training
400      4K      40K     80K
         ↓       ↓       ↓
Wan2.2†: 56.26% → 56.44% → 60.25% → 60.98%
Wan2.1†: 56.80% → 60.25% → 62.61% → 63.34%
SVD†:    56.80% → 60.25% → 60.98% → 60.98%
```

### 7.3 关键发现

1. **持续改进**：更多后训练数据持续提升AR性能
2. **模型规模效应**：更大的模型（14B）比小模型（1.5B）从更多数据中获益更多
3. **预训练 vs 后训练**：动作条件的后训练比升级预训练视频生成器更有效

### 7.4 启示

- **数据收集优先**：对于具身任务，收集目标领域的动作-观察数据比使用更大规模的预训练数据更有效
- **Scaling方向**：应该扩展动作条件的后训练数据，而非仅仅增加网络视频预训练数据

## 8. 推理时间 Scaling

### 8.1 实验设置

研究推理时分配的计算资源如何影响性能：
- 平均推理次数（模拟的潜在未来数）：3到11
- 评估任务：Active Recognition
- 模型：Wan2.1†, SVD†

### 8.2 结果

```
Avg Inference Count per Episode
3       4       5       6       7       8       9       10      11
         ↓       ↓       ↓       ↓       ↓       ↓       ↓       ↓
Wan2.1†: 53.36% → 56.44% → 57.17% → 56.62% → 58.26% → 59.71% → 62.61%
SVD†:    53.36% → 56.44% → 57.17% → 60.98%
```

### 8.3 关键发现

1. **正向相关**：增加推理时的模拟次数与成功率呈明显正相关
2. **显著提升**：推理次数从3增加到11，SR从53.36%提升到60.98%
3. **计算权衡**：更多推理带来更好的决策，但也增加计算成本

### 8.4 启示

- **推理时间训练**：分配更多推理时间计算来模拟潜在未来，使规划器能够做出更明智的决策
- **资源分配**：为关键决策步骤分配更多推理资源
- **新研究方向**：探索自适应推理资源分配

## 10. 未来研究方向

### 10.1 增强泛化能力

#### 10.1.1 问题

当前世界模型在新环境中泛化能力有限。

#### 10.1.2 方向

- 统一动作表示
- 课程学习
- 领域特定数据收集
- 更强大的预训练策略

### 10.2 长视界规划

#### 10.2.1 问题

当前模型难以支持长视界连贯规划。

#### 10.2.2 方向

- 空间记忆（spatial memory）
- 情节记忆（episode-level memory）
- 更好的长期依赖编码和检索

### 10.3 精确物理建模

#### 10.3.1 问题

操作任务中的接触丰富交互难以准确建模。

#### 10.3.2 方向

- 物理引导的运动生成
- 物理特性推断和生成
- 将物理信号整合到条件路径中

### 10.4 更强的策略

#### 10.4.1 问题

提议和修订策略的性能直接影响整体表现。

#### 10.4.2 方向

- 更强大的策略模型
- 深化世界模型和决策的协同
- 更符合人类偏好的奖励模型

### 10.5 真实世界部署

#### 10.5.1 方向

- 在真实机器人上评估
- 处理真实世界的噪声和不确定性
- 实时性能优化

## 11. 总结与思考

### 11.1 核心贡献总结

World-in-World是首个在闭环环境中评估生成世界模型的开放平台，其核心贡献包括：

1. **统一的闭环规划框架**：提出-模拟-修订的三阶段决策流程
2. **标准化的动作API**：支持异构世界模型的集成
3. **全面的基准测试**：涵盖四个具身任务的系统评估
4. **数据Scaling Law**：首个关于世界模型训练数据规模的量化研究
5. **推理时间Scaling**：揭示推理计算资源分配的价值

### 11.2 对Spatial AGI的意义

World-in-World对Spatial AGI的发展具有重要意义：

1. **评估范式转变**：从开环视觉质量评估到闭环任务成功评估
2. **关键因素识别**：可控性比视觉质量更重要
3. **训练策略指导**：动作条件的后训练比纯视觉预训练更有效
4. **资源分配启示**：推理时间计算可以显著提升性能

### 11.3 开放问题

1. 如何提升世界模型在新环境中的泛化能力？
2. 如何支持长视界的连贯规划和决策？
3. 如何精确建模物理交互和动力学？
4. 如何在保持效率的同时提升决策质量？

### 11.4 思考与展望

World-in-World为世界模型和Spatial AGI的研究提供了重要的新视角。论文的核心发现——**可控性优先于视觉质量**——值得深思。这表明：

- 生成逼真内容不等于具备智能决策能力
- 动作-观察的闭环交互是检验智能的关键
- 数据质量和任务相关性比数据规模更重要

未来，我们期待看到更多关于闭环评估、推理时间Scaling、以及真实世界部署的研究。World-in-World为这一方向奠定了重要基础。
